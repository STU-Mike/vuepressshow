# nodejs 模块系统知识点

# 为什么要模块系统

是为了让nodejs的文件可以互相调用。

# 模块是什么

模块是Node.js应用程序的基本组成部分，文件和模块是一一对应的，也就是说一个Node.j文件就是一个模块.

# 创建模块

nodejs提供了exports和require两个对象，

  1. exports：是模块公开的接口
  2. require：用户从外部获取一个模块的接口，即获取模块的exports对象 例子代码 下面是用require（'./Hello'）引入当前目录下的hello.js文件（其中./为当前目录，node.js默认后缀为js）
var hello = require('./hello');
hello.world();

下面是hello.js文件的内容

exports.world = function() {
  console.log('Hello World');
}

上面示例中，hello.js通过exports对象吧world作 为模块的访问接口，在main.js中通过require('./hello')加载这个模块，然后就可以直接访问hello.js中exports对象的成员函数了。

# 把一个对象封装到模块中

简单来说就是把这个对象赋值给export（模块的接口） 格式如下

module.exports = function() {
  // ...
}

例子

//hello.js 
function Hello() { 
    var name; 
    this.setName = function(thyName) { 
        name = thyName; 
    }; 
    this.sayHello = function() { 
        console.log('Hello ' + name); 
    }; 
}; 
module.exports = Hello;

注意module（模块）.exports（接口） = hello ，就是把hello赋值给了接口， 使用：

//main.js 
var Hello = require('./hello'); 
hello = new Hello(); 
hello.setName('BYVoid'); 
hello.sayHello();

模块接口的唯一变化是使用module.exports=Hello 代替了exports.world = function(){}。 在外部引用该模块时，其接口对象就是要输出的Hello对象本身，而不是原先的exports

exports 和 module.exports 的使用 如果要对外暴露属性或方法，就用 exports 就行，要暴露对象(类似class，包含了很多属性和方法)，就用 module.exports。

# 服务端的模块放在哪里

之前我们就已经使用过模块了 比如： var http = require("http");

...

http.createServer(...); nodejs中自带了一个叫做http的模块，我们在我们的diamante中请求它并返回值赋给一个本地变量。这把我们本地变量变成了一个拥有所有http模块所提供的公共方法的对象。 nodejs的require方法中的文件查找策略如下 nodejs有四类模块：原生模块和3中文件模块 各个模块的加载优先级不同 优先级：文件模块缓存区>原生模块缓存区>未缓存的原生模块>文件模块

# 文件模块缓存

优先级最高

# 原生模块缓存区

优先级次于文件模块换缓存。require方法在解析文件名之后，优先检查模块是否在原生模块列表中，以http模块为例，尽管在目录下存在一个http/http.js/http.node/http.json文件，require（”http“）都不会从这些文件中加载，而是从原生模块中加载 原生模块也有一个缓存区，同样也是优先从缓存区加载，如果没有被加载过，则调用原生模块的加载方式进行加载和执行。

# 从文件加载

当文件模块缓存中不存在，而且不是原生模块的时候，nodejs会解析requeire方法传入的参数，并从文件系统中加载实际的文件， require方法接受几种参数的传递

  • http、fs、path等，原生模块
  • ./mod或../mod,相对路径的文件模块
  • /pathtomodule/mod,绝对路径的文件模块
  • mod，非原生模块的文件模块。 ##全局变量 _dirname:表示当前执行脚本所在的目录 _filename:表示当前正在执行的脚本的文件名。它将输出文件所在位置的绝对路径。如果在模块中，返回的值是模块文件的路径。
